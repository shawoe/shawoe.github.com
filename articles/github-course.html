GitHub：使用指南

  • 作者：肖艺
  • 联系方式：shawoe@163.com

大家一直在聊的 GitHub Pages 是什么？

如果你对编程有所了解，就一定听说过GitHub。它号称程序员的Facebook，有着极高的人气，许多重要的开源项目都托管在上面。简单说，它是一个具有版本管理功能的代码仓库，而且每个项目都有一个主页，列出项目的源文件。

但是对于一个新手来说，看到一大堆源码，只会让人头晕脑涨，不知何处入手。他更希望看到的是，一个简明易懂的网页，说明每一步应该怎么做。因此，GitHub就设计了Pages功能，允许用户自定义项目首页，用来替代默认的源码列表。所以，GitHub Pages可以被认为是用户编写的、托管在GitHub.com上的静态网页。

而GitHub for Windows则是Windows上的客户端软件，用于本地仓库和远程仓库的同步。当然了，在介绍使用之前还得先啰嗦上几句：

这是我在GitHub上写的第一篇博文，怀着非常忐忑的心情，毕竟是如此高大上的代码托管仓库，要写好真心不容易。查看网上其他GitHub指导性博客时，着实发现大部分的操作都已经跟不上GitHub当前版本了，这样让很多人使用GitHub带来了不便。于是便萌生了重新整理《GitHub：入门指南》的想法。但毕竟本人水平有限（甚至毫不夸张的说还只是刚刚入门的水平），文中有任何不当之处请读者谅解并指出。毫无疑问，随着时间的推移这篇博文也终将不适应新版本的GirHub，但我想至少在当下还是具有一定参考价值的。

需要注意的是，这篇博文并非完全本人原创，它是针对GitHub Pages和GitHub for Windows两个常用工具，并参考另外两位技术大牛的博文整理而成，最终以“迭代”这一互联网最传奇的方式将此文用GitHub for Windows上传至GitHub Pages。文中部分摘抄内容如有冒犯原作者之处希望谅解并联系本人及时删改。下面是我参考的两篇博文：

  • http://www.cr173.com/html/15618_1.html
  • http://blog.csdn.net/renfufei/article/details/37725057

接下来，我们一起来试着使用GitHub Pages服务并安装GitHub for Windows。

  1. 注册账号

    2. 首先，进入GitHub主页，地址:https://github.com在右侧输入账号、邮箱、密码，然后点击注册按钮。

  2. 初始设置

    3. 注册完成后,选择Free免费账号完成设置。（如果要求填写银行卡号，留空即可。）

  3. 验证邮箱

    4. 请打开你的邮箱，查看发送给你的确认邮件，你需要验证邮箱后，后面生成的个人主页才会被接受和发布。打开确认邮件中的地址，点击你的邮箱“confirm”，验证成功后再次进入GitHub主页：https://github.com如果你成功看到如下页面，说明你已经注册成功了。

  4. 创建仓库

    5. 经过上面的步骤，你的GitHub主页地址是:https://github.com/shawoe这个仓库的名字是和你的账号对应的，比如我的是：https://shawoe.github.io 然后在GitHub主页点击右下角的创建仓库（即：+New repository），输入基本信息，然后再次点击创建仓库。

  5. 进入项目设置页面

    6. 因为这个项目就是专门的放页面的，所以你会看到banch:master，这便是你的默认分支，能显示你的文件和代码。后面的步骤会告诉你如何创建 gh-pages 分支。

  6. 进入自动页面生成器

    7. 在设置页面，下拉到底部，默认分支(master)不用管，稍后我们会修改它。

  7. 创建用户页面

    8. 输入一些内容，可以自己编辑，也可以从readme加载，当然你也可以跳过这一步。

  8. 继续，选择主题

    9. 在设置页面，下拉到底部，默认已经写好的主页代码可以不用管，当然你也可以自己去写（如果你不累的话），点击继续。

  9. 选择主题然后发布

    10. 如下图所示，其实这些以后你都可以自己修改替换，这只是生成一些css，html，img到你的仓库里面。

  10. 查看效果

    11. 现在，你可以访问自己的GitHub主页了，地址是：https://shawoe.github.io/shawoe注意，地址开头那个是你的用户名，末尾那个是你的仓库名。再举一个例子，我在GitHub上的电子简历页面地址是：https://shawoe.github.com/resume总之，以此类推吧，刚刚创建的GitHub Pages效果如下图所示:

  11. 设置默认分支

    12. 到这里GitHub Pages就已经创建完成了，不过为了更方便下面的操作，最好还是修改一下默认分支。如下图所示，重新打开Setting后将默认分支设置为gh-pages：

  12. 下载GitHub客户端

    13. 对于习惯Windows图形界面的程序员来讲，GitHub的使用是需要点时间和耐心的，然而最近GitHub发布了GitHub for Windows 则大大降低了学习成本和使用难度，他甚至比SVN都简单，好吧，你不信，我们来一步一步过一面：

  13. 安装GitHub客户端

    14. 首先我们需要下载GitHubSetup，进入下面的地址：https://windows.github.com点击下载GitHub for Windows，然后安装就不说了，因为是在线安装需要比较长的时间请耐心等待，进入安装界面后凭你的直觉点下一步就好了。

  14. 登陆GitHub客户端

    15. 安装好后打开GitHub，用你的GitHub账号登陆。需要注意的事，GitHub for Windows会帮你创建SSH Key，你打开邮件应该会收到邮件帮你创建SSH Key的信息，这也省了不少事：

  15. 复制仓库

    16. 点击左上角的那个大大的加号按钮，选择右边的"Clone"将之前在GitHub上面创建的仓库复制到本地。

  16. 创建仓库

    17. 当然了，你也可以直接创建一个新的仓库，输入相关信息：

    但是要注意，新创建的仓库应重复上面的5-11步给新仓库创建gh-pages分支。

  17. 公布仓库

    18. 这个时候我们就可以Publish一个仓库了（如果是Clone的仓库请跳过此步骤），Publish之后按钮就变成sync文字，表示以后的修改提交就是同步，这个时候你到你的GitHub上看应该就会多一个仓库了：

  18. 添加项目

    19. 仓库在本地保存在一个文件夹中，GitHub for Windows会监视这个文件夹，任何修改都会在仓库中看到，我们可以把编写好的物业或者源代码放入这个文件夹下，当然，你直接在文件夹下编辑文本也是可行的。

  19. 提交注释到本地仓库

    20. 将修改提交到本地仓库，回到GitHub for Windows中，已经可以看到右侧有你刚才添加的项目了，那么在中间“Uncommitted change”填写摘要和注释，目的只是记录一下你做了哪些操作：

  20. 将本地仓库提交

    21. 最后，点击"sync"将本地仓库同步提交到GitHub。

  21. 在Github上查看你的项目

    22. 如下图所示，打开后你将看到成功开源了的项目：

  22. 参与开源项目协同开发

    23. 要知道GitHub现在已经深度支持Windows开发者了，现在每个在GitHub的开源项目右下角都有一个“Clone in Desktop”的按钮，我们可以Clone任何我们感兴趣的开源项目，然后在本地进行修改，参与开源项目的开发。点击该按钮之后，浏览器会要求打开GitHub for Windows，并下载项目至本地，新建一个本地仓库。比如说下面就是一个我暑期写的轻量级Java EE开源应用，我们可以将它Clone至本地：https://github.com/shawoe/javaweb
